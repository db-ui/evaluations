New icon font with ligatures

New icon font with ligatures, current stroke algorithm:

Benefits

  • Referencing doesn't need a mapping like when using glyphs, that additionally needs to get in sync in between solid and outline
  • We could set the icons stroke by text-stroke from the outside
  • Instead of using SVGs, referencing these icons assets (fonts vs. SVGs) is much simpler in all of our usecases

Original
Original image, only manipulated by removing width and height attributes from within the SVG file. So those are scaling linear and even also don't have the vector-effect="non-scaling-stroke"-attribute defined within.
Regular
Same as original, but additionally with the dynamic stroke-width (stroke-width: var(--stroke-width, 2)) and attribute vector-effect="non-scaling-stroke" defined within the SVG files.
Regular - stroke to fill
Same as Regular, additionally converted stroke to fill.
Optimized
Same as Regular, but additionally run through ImageOptim.
Optimized - stroke to fill
Same as Optimized, additionally converted stroke to fill.
